# Contributing

Thanks for your interest in contributing to Anymon! Small, focused pull
requests are easiest to review.

Guidelines

1. Fork the repository and create a feature branch.
2. Follow the code style: run `cargo fmt` before committing.
3. Add or update tests for significant behavior changes.
4. Update documentation in `docs/` or the root `README.md` when adding
   features or changing behavior.
5. Ensure `cargo test` passes locally.

Pull requests

- Provide a clear description and motivation for the change.
- Link any relevant issues or discussion threads.

Code of Conduct

Please follow the `CODE_OF_CONDUCT.md` when interacting in issues and PRs.
